# Meno

A glass widget card for Android — analog clock, digital time, date, weather and battery — living on the home screen as an almost-invisible superellipse of glass.

Every part of this project — graphics, layout, bytecode, packaging, signing — was generated by AI (Anthropic Claude Opus 4.8) through plain conversation. Not a single line was written by a human hand. No Android Studio, no Gradle, no Java source: hand-authored **smali** bytecode and binary resources, compiled with `apktool` + `aapt` and signed with `apksigner`.

**Package:** `io.github.shumtugle.meno` · **Version:** 1.9.10

## What it shows
- **Analog clock** — the hands are moved by Android itself, so the widget costs no battery
- **Digital time & date** — formatted in the app language, not the system's
- **Weather** — sky icon + temperature from [Open-Meteo](https://open-meteo.com), by your location, cached and repainted on every full frame
- **Battery** — read locally, no permission

## Touch
- **Weather pill** — a hidden tap zone at its left edge forces a weather refresh; the rest of the pill opens a web weather search
- **Clock → alarms · Date → calendar · Battery → battery usage**
- Any of the five zones can be re-pointed to any app through the system picker, and reset with one tap

## Adapts to any launcher
The layout rides on weights, not fixed widths — it fills a wide cell and shrinks into a narrow one without clipping. It refreshes itself the moment a launcher places or resizes it (`onAppWidgetOptionsChanged`), so weather appears without any manual nudge.

## Customization (in-app Settings)
- Background: dimming (10–90 %), gradient direction (▲▼◀▶⤢), or full transparency
- Second-hand colour — seven options, via stacked `AnalogClock` layers
- Time format: auto / 24 h / 12 h
- Language — fourteen flags on the first screen, or follow the system

## Permissions
- **Internet** — to fetch the weather; nothing else leaves the phone
- **Location (coarse)** — weather where you are; optional, falls back to a default point when off

## Structure

This is **not** a Gradle project. The source is a hand-authored tree under [`src/`](src/):

```
src/
├── AndroidManifest.xml       binary manifest (decoded)
├── apktool.yml               build metadata (version, sdk)
├── smali/io/github/shumtugle/meno/    the logic, as Dalvik bytecode
│   ├── MenoWidget.smali        the widget provider
│   ├── WeatherJob.smali        weather fetch, cache, repaint poke
│   ├── ConfigActivity.smali    the settings panel
│   ├── AboutActivity.smali     the about screen
│   └── LocaleUtil.smali        manual language switching
└── res/                      layouts, drawables, strings, icons
```

## Building

```bash
apktool b src -o out.apk
# Android 11+ requires resources.arsc stored uncompressed:
cd pack && unzip -o -q ../out.apk -x "META-INF/*"
zip -X -0 ../clean.zip AndroidManifest.xml resources.arsc
zip -X -r ../clean.zip classes.dex res
cd .. && zipalign -f 4 clean.zip aligned.apk
apksigner sign --ks your.keystore --v2-signing-enabled true --v3-signing-enabled true \
  --min-sdk-version 26 --out meno-1.9.10.apk aligned.apk
```
